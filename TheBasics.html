What is an API?

APIs (Aplication Program Interface) and what they provide, can vary greatly from host to host. Generally, they provide information or functionality that the host has developed or owns. They then allow you to access and use it on your project via their API setup. It has clearly defined protocols and tools to shape how you interact with and access thair data. APIs provide the programmer of an application or website an easy way to use pre-defined information and functionality, saving the developer a lot of time. The Petfinder API gives you access to their expansive database, as well as many options for gathering and using the information. We will get into specifics in a little bit.

If you would like to delve deep into the API specifics, here is a 6-part explanation.

RESTful API

The Petfinder API is a RESTful API. The structure of a RESTful API is based on REST architecture style. This style is defined by six constraints that are required in order for an API to be RESTful:

  • Client-Server:

    Separates user interface concerns from storage concerns. This means that the user interface and the server can be altered and scaled independently of eachother. If the database you are accessing changes, while you may get new data, your interface is not affected.

  • Stateless:

    No client context is stored on the server between requests during client-server comminication. Each client request contains all necessary information, and the state of the session is held in the client, not the server. The client sends requests when it is ready to transition into a new state. If the session must be in a persistent state, such as for authentication, the session state can be transderred to a different server.

  • Cacheable:

    Responses can be cached. Clients must define their responses as either cacheable or not in order to prevent the use of stale or inappropriate data. This feature improves overall ability and performance.

  • Layered System:

    A client can't tell if it is in contact with one main server, or intermediary servers. This provides many benefits:

      Improves system scalability.
