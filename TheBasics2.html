GET method, JSON and jQuery

GET Method

To communicate with the Petfinder API we must use the GET method. GET requests send the information needed by the target, usually a server, in the URL by appending it to the end. Whether you know it or not, you have seen this many times. If you didn’t know what to look for it is very likely it just looked like random garbled code. Now it is important that you understand what this seemingly garbled code stands for because this is how you interact with the API.

The URL

URL stands for Uniform Resource Locater and it is the address of the resource you are requesting. The URL can be broken down into parts, each part getting more and more specific until it maps to exactly what you want. Similarly, if someone wants to send you a package, they need to know all the details that will route the package to your house. They need your home address, which is comprised of your country, state, city, street, house number, and possibly apartment number. If they are missing any part, aside from the country, it is very likely you won't get your package. The same goes for the URL. If you are missing any part of it, aside from the HTTP, you probably aren’t going to get where you are trying to go.

Let's take a look at the sample code from the Petfinder API documentation:

http://api.petfinder.com/my.method?key=12345&arg1=foo&token=67890&sig=abcdef

  • http: (HyperText Transfer Protocol) This is the request-response protocol between the client and server.

    When the internet was new, this always needed to be in front of the URL or else it wouldn't work. Now, it would be a safe bet that you never write it in front of a URL you are typing from memory, yet it still works. This is because it can be assumed that every URL is using HTTP, the same as a letter sent within the United States without writing USA will still get there. If you look at the rest of the address format, there are hints, such as the state it is going to, that imply its destination is in this country. URLs all follow a precise format which implies it is following correct protocol.

    Note: If you want to add a level of security HTTPS should still be included.

  • api.petfinder.com At the level of understanding needed for this guide, this is pretty self explanatory.

  • my.method Directions to the destination. The path starts with a single slash (never double), and any subsequent segments are separated slashes. The segments are usually in a specific hierarchical order like the file path on your OS. While the path may be empty (no extra information after the domain), it is always defined. Finding your way to the main page with nothing more than the domain name is considered following a path.

  • ?key=12345&arg1=foo&token=67890&sig=abcdef represents the information being transferred to the server in a GET request. Most parameters are formatted as name=info with & between each set of values.

JSON

JSON stands for JavaScript Object Notation. It is a data syntax. When requesting data from the API, all the data must be sent in text form. Communications with this API can be in XML or JSON format. XML is a little dated and JSON is more widely used now, so we will be sticking with JSON. Javascript objects can easily be changed into JSON, and vice versa, making sending and using data simple.

JavaScript and JSON objects are very similar, and it is easy to change from one type of object to another. A JSON object is basically a JavaScript object converted into a string. JSON.stringify() will turn a JS object into a JSON string object and JSON.parse() turns the JSON string into a JavaScrpt object.

jQuery

jQuery is a JavaScript library, and it exists to make your life easier. It is used extensively all over the internet.

jQuery is most likely used on the site of any company you can think of. It is extremely popular and all major browsers support it. jQuery $.get() Method and AJAX

As you know, the GET method requests data from the server via a GET request. The jQuery $.get() method is no different, it just has different syntax than you may be used to. The example above didn’t use JavaScript because there was no need to complicate the concept with syntax we won’t be working with.

$.get() is an AJAX method. To put it simply, AJAX is a way to communicate back and forth with the server, potentially updating parts of the page the user is interacting with without needing to update the entire page. We could go into a lot more detail about AJAX and its functionality, but for our purposes that is about all we need to know.
