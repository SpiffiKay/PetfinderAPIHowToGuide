GET method and JSON

GET Method

To access the Petfinder API we have to use GET requests, which sends the target information via the URL.

The URL

URL stands for Uniform Resource Locater. It is the address of the resource you are requesting. The URL can be broken down into basic parts, and we will be building URLs to request information from the API, so it is important to understand each part. Much like your home address, which is comprised of your city, state, street, house number, and possibly apartment numbers, the URL is also comprised of various parts, which all together leads to your destination.

http://api.petfinder.com/my.method?key=12345&arg1=foo&token=67890&sig=abcdef

  • http: Hypertext Transfer Protocol: This is the request-response protocol between the client and server.
  • api.petfinder.com Domain name. This is pretty self explanatory.
  • my.method Path: Directs to a specific file or directory on the server.
  • ?key=12345&arg1=foo&token=67890&sig=abcdef Query: represents the information being transferred to the server in a GET request. Each parameter is formatted as name=info with & between each set of values. For example key=12345 is representative of your API key.

JSON

JSON stands for JavaScript Object Notation. It is a data syntax. When requesting data from the API, all of the data must be in text form. Javascript objects can easily be changed into JSON.

JavaScript and JSON objects are very similar, and it is easy to change from one to another.
